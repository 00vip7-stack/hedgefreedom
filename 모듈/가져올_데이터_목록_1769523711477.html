가져올 데이터 목록

1. 어떤 데이터를 가져와야 하나? (핵심 항목) 단순한 회계 장부가 아니라 '자금 수지(Cash Flow) 데이터'를 가져와야 합니다. 외화 매출/매입 채권 데이터: 인보이스(Invoice) 발행 기준 결제 예정일. 통화종류(Currency): USD, EUR, JPY 등 (통화가 같아야 매칭 가능). 금액(Amount): 입금액과 출금액. 확정 여부: 실제 입금 확인된 건인지, 예정 건인지 구분. 2. API로 가져오는 방법 (더존 기준) 더존은 현재 더존 개발자 센터(Developers)를 통해 API를 제공합니다. 방법: 귀하의 서비스가 더존의 파트너로 등록되면, 고객(기업)의 승인을 얻어 '미수금/미지급금 현황'이나 '자금일보' 데이터를 JSON/Excel 형식으로 호출할 수 있습니다. 자동화: 사용자가 매번 엑셀을 다운로드해서 업로드할 필요 없이, 귀사의 서비스가 매일 새벽에 API로 데이터를 동기화하여 아침마다 "오늘 날짜가 겹치는 내역입니다"라고 리포트를 쏴주는 것이 핵심 경쟁력입니다. 3. "날짜가 겹친다"는 것의 알고리즘 (로직) 엑셀로 데이터를 받았다면 시스템은 다음을 수행합니다. 동일 날짜 매칭: 2026년 2월 10일 입금 $10,000 / 출금 $8,000 → "이날은 $2,000만 환전하면 됩니다"라고 안내. 인접 날짜 매칭 (가장 중요): 2월 10일 입금 / 2월 12일 출금 → 이틀 차이는 외화 통장에 잠시 거치하면 되므로 매칭 가능 범위로 묶어줍니다. (이걸 '버퍼 설정'이라고 합니다.) 부분 매칭: 입금액이 출금액보다 적을 경우, 부족분만 선물환 거래를 하도록 금액을 쪼개줍니다. 4. 저가형 서비스를 위한 전략적 제언 더존 API 연동이 처음에는 비용이나 절차가 부담스러울 수 있습니다. 이때 가장 저렴하게 시작하는 순서는 다음과 같습니다. 1단계 (가장 저렴): 사용자가 ERP에서 내려받은 엑셀 파일을 그대로 업로드하면, 귀사의 웹사이트에서 '매칭 결과'만 예쁘게 보여주는 방식 (개발비 최소화). 2단계 (자동화): 이카운트(Ecount)처럼 API가 개방적이고 저렴한 ERP부터 우선 연동하여 "자동 동기화" 기능을 홍보. 3단계: 어느 정도 사용자가 모이면 더존, SAP 등과 정식 파트너십을 맺고 고도화. 결론: 데이터를 가져오는 것 자체는 어렵지 않습니다. 다만 "ERP마다 제각각인 엑셀 형식을 우리 시스템이 얼마나 똑똑하게 읽어내어(파싱), 실무자가 손대지 않아도 매칭 결과를 보여주느냐"가 저가형 서비스의 승부처가 될 것입니다. 더존 비즈온 API 안내 등을 통해 연동 가능한 데이터 범위를 먼저 확인해 보시는 것을 추천합니다.
